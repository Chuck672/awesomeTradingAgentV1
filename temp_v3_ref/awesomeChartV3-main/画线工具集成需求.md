# 项目需求说明书：基于 Lightweight Charts 1:1 复刻 TradingView 绘图工具

## 1. 项目概述
本项目旨在基于 `lightweight-charts` (版本需 >= 4.0.0) 的 `ISeriesPrimitive` (Plugin API)，从零构建一套完整的、支持交互的图表绘图工具系统。目标是达到与 TradingView 原生绘图工具相似的 1:1 交互和视觉体验。

**需要实现的工具列表**：
1. Arrow (箭头线)
2. Trendline (趋势线)
3. Horizontal Ray (水平射线)
4. Horizontal Line (水平线)
5. Rectangle (矩形)
6. Long Position (多头头寸)
7. Short Position (空头头寸)
8. Measure (测量尺)

## 2. 核心架构设计 (绝对约束)

由于 LWC 原生不支持拖拽画图，**必须采用以下架构**：

1. **坐标系存储基准**：所有图形的锚点（Points）必须以 `{ time: Time, price: number }` 的逻辑坐标进行存储，**绝对不能存像素坐标**，以保证图表拖拽、缩放时图形能完美跟随 K 线。
2. **渲染机制**：使用 LWC v4 的 `ISeriesPrimitive` 接口。每个工具作为一个独立的 Primitive 实例，挂载到 K 线 Series 上。
3. **事件拦截层 (Event Overlay)**：由于 LWC 原生 API 无法很好地处理 `mousedown -> mousemove -> mouseup` 拖拽连贯操作，必须在图表容器 (Container) 上层覆盖一个 `z-index` 更高的透明 `div` (或直接监听 container 的原生 DOM 鼠标事件)。
4. **坐标转换**：在鼠标事件中，通过原生 `offsetX/Y`，结合 `series.coordinateToPrice()` 和 `chart.timeScale().coordinateToLogical/Time()` 进行像素与业务逻辑数据的双向转换。

4. 全局状态机 (DrawingManager.ts)
实现交互的核心，管理以下状态：

模式: Idle (空闲浏览模式), Drawing (正在绘制), Editing (选中并调整图形)。
交互流程:
用户在外部 UI 点击某个工具（例如 Trendline）。Manager 进入 Drawing 准备态。
用户在图表上 点击第1下：实例化对应的 Tool 类，记录起点 p1 (time, price)。
用户 移动鼠标：实时更新终点 p2，强制 LWC 重绘 (requestUpdate)，实现跟随鼠标的动态预览。
用户 点击第2下：确立终点 p2，当前工具绘制完成，存入工具列表，Manager 切回 Idle 模式。
5. 各工具具体逻辑与 Canvas 实现方案
所有工具的 Renderer 类必须在 draw(target: CanvasRenderingContext2D, priceScale, timeScale) 方法中，将 time/price 转换为当前屏幕的 x/y 像素再进行绘制。

5.1 Trendline (趋势线)
锚点: 2个点 (p1起点, p2终点)。
绘制逻辑: ctx.moveTo(p1.x, p1.y) -> ctx.lineTo(p2.x, p2.y) -> ctx.stroke()。
交互: 点击2次完成。
5.2 Arrow (箭头线)
锚点: 2个点 (p1起点, p2终点)。
绘制逻辑:
画出 p1 到 p2 的 Trendline。
几何计算: 计算线段斜率 Math.atan2(p2.y - p1.y, p2.x - p1.x)。在 p2 处利用正余弦函数，向后反算一个底边为 w，长为 l 的等腰三角形的三个顶点。
使用 ctx.beginPath(), lineTo(), ctx.fill() 绘制箭头盖在 p2 上。
交互: 点击2次完成。
5.3 Horizontal Line (水平线)
锚点: 1个点 (p1, 仅依赖 price，忽略 time)。
绘制逻辑: Y坐标固定为 p1.price 转换的像素Y。X 坐标从 0 到 canvas.width。贯穿整个屏幕。
交互: 点击1次即可完成。
5.4 Horizontal Ray (水平射线)
锚点: 1个点 (p1，需要 time 和 price)。
绘制逻辑: 起点为 p1 的 (x, y)。终点为 (canvas.width, p1.y) (默认向右侧无限延伸)。
交互: 点击1次完成。
5.5 Rectangle (矩形)
锚点: 2个点 (p1 左上角/或任意一角, p2 右下角/或对角)。
绘制逻辑:
计算 width = Math.abs(p2.x - p1.x)，height = Math.abs(p2.y - p1.y)。
计算左上角真实的像素起点 startX = Math.min(p1.x, p2.x)，startY = Math.min(p1.y, p2.y)。
绘制半透明的 ctx.fillRect 和实线的 ctx.strokeRect。
交互: 点击2次完成。
5.6 & 5.7 Long Position / Short Position (多/空头头寸)
锚点: 2个时间点 (startX, endX)，3个价格点 (EntryPrice, TakeProfitPrice, StopLossPrice)。
交互逻辑:
点击第1下确定 EntryPrice 和 startX，默认根据图表当前比例赋予一个初始的 TakeProfit 和 StopLoss 高度。
此工具较复杂，需要绘制两个相连的矩形（多头：上方绿底，下方红底；空头反之）。
绘制逻辑 (以 Long 为例):
Y_Entry = entryPrice; Y_TP = takeProfitPrice; Y_SL = stopLossPrice。
画上方矩形 (Profit区): Y坐标从 Y_TP 到 Y_Entry，颜色半透明绿。
画下方矩形 (Loss区): Y坐标从 Y_Entry 到 Y_SL，颜色半透明红。
文字绘制: 在中间线 (Entry) 画文本显示价格；在上下区域中心写字显示 "盈亏比 (Risk/Reward Ratio)" 和具体的 PnL 点数。
5.8 Measure (测量尺)
锚点: 2个点 (p1, p2)。
行为特殊性: 在 TradingView 中，Measure 释放鼠标后如果是空白处通常会消失，或者固定在屏幕上直到下一次点击。为简化，设定为：按住鼠标拖拽显示，松开鼠标完成并固定，按下 Esc 或下次画图时清除。
绘制逻辑:
绘制一个覆盖 p1 和 p2 的半透明蓝色/灰色 Rectangle。
绘制 p1 到 p2 的箭头连线。
计算指标: 计算两点之间的 K线根数 (p2.logicalIndex - p1.logicalIndex)，价格差 (p2.price - p1.price)，价格百分比变化。
使用 ctx.fillText 在矩形中心偏上方绘制带背景的文本框，显示上述计算出的文本数据。
6. 开发执行步骤 (建议 Trae 分步骤执行)
Step 1: 基础设施搭建

初始化 Vite + TS 项目，安装 lightweight-charts。
构建原生的 DOM 鼠标事件监听系统 (DrawingManager)，完成 Canvas 像素坐标到 Logical(Time)/Price 的相互转换打通。
Step 2: 核心基类实现

实现 BaseDrawingTool 及其 LWC 要求的 PaneView 和 PaneRenderer 接口。确保在 Renderer 中能够拿到正确的 Context 和坐标缩放 API。
Step 3: 基础线段类工具开发

依次实现 Trendline, Arrow, Horizontal Line, Horizontal Ray。测试它们的连续点击绘制过程。
Step 4: 形状与复杂区域工具开发

实现 Rectangle 和 Measure (涉及到宽高计算和文本绘制)。
Step 5: 高级金融工具开发

实现 Long/Short Position。这部分要求 UI 精美，需要计算 Risk/Reward 比例并展示在 Canvas 上。

7. 代码规范要求
严格使用 TypeScript 强类型，定义好所有的 Point 接口。
由于在 mousemove 中会频繁调用 requestUpdate 导致 Canvas 重绘，代码中切勿在 draw 方法里进行复杂的大型数组遍历或创建无用的对象，避免内存泄漏和掉帧。
保证颜色、线条粗细 (lineWidth) 通过 Options 接口暴露，方便后续自定义。
