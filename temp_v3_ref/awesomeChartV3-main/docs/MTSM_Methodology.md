# MTSM (Market Structure Targets Model) - 核心逻辑实现文档

基于最新提供的图表与规则，并结合程序化实现的边缘情况（回撤确认深度、极端趋势过滤、冷启动等），MTSM 指标的逻辑从传统的固定周期极值（ZigZag）变更为**基于价格突破与深度回撤确认的动态市场结构追踪**。

以下是程序化实现所需的逻辑拆解：

## 1. 核心概念 (Core Concepts)
* **HH (Higher High)**: 更高的高点
* **HL (Higher Low)**: 更高的低点
* **LH (Lower High)**: 更低的高点
* **LL (Lower Low)**: 更低的低点
* **MSB (Market Structure Break)**: 顺势结构突破。在多头中突破 HH，在空头中突破 LL。
* **SHIFT (Market Structure Shift)**: 逆势结构破坏（反转）。在多头中跌破最近的 HL，在空头中突破最近的 LH。
* **GL (Grab Liquidity) / Liquidity Break**: 流动性猎取/假反转。在发生 SHIFT 之后，价格没有延续反转趋势，而是回头突破了引发 SHIFT 的起点（多头的 HH 或空头的 LL）。

---

## 2. 空头趋势规则 (Bearish Trend)

### 2.1 LL (Lower Low) 的确认条件
1. **突破**: 价格必须向下跌破前一个 LL。
2. **回撤确认 (Retracement Confirmation)**: 价格在跌破前一 LL 后，必须向上产生足够的反弹回撤。**程序化定义**：回撤深度必须 **>= `(前一LH - 前一LL) * 0.1`**。
3. **标记**: 当回撤条件满足后，这波向下突破的绝对最低点被正式标记为新的 LL。

### 2.2 LH (Lower High) 的确认条件
1. **相对高度**: LH 的顶点必须是最靠近前一个 LH，并且没有突破它（即低于前一个 LH）。
2. **极值过滤 (Pivot Filter)**: 废除“必须在前一个LL(1)上方”的绝对高度限制。**程序化定义**：只要价格在形成新的 LL 之前，向上反弹产生了一个局部的 ZigZag 极值高点（Pivot High），且满足回撤确认深度的要求，即可作为候选 LH。这能有效应对单边暴跌（瀑布行情）中反弹微弱的场景。
3. **突破确认**: 当价格再次向下跌破最新的 LL 候选点（即发生 MSB）时，这段回撤的最高点（Pivot High）才被正式确认为 LH，并在此处画出 MSB Level 突破线。

### 2.3 空头 SHIFT 的确认条件
1. **反转突破**: 价格从 LL 顶点开始反转，并且最终向上突破了**最近的一个确定的 LH** 顶点。
2. **结构成熟度**: 在发生 SHIFT 时，之前必须已经存在 **两对或以上** 的 (LH + LL) 结构。

### 2.4 空头 GL (Grab Liquidity)
1. **发生条件**: 在空头趋势发生 SHIFT（向上突破 LH，理论上转多）之后，价格并没有继续上涨创造 HH/HL，而是向下 **跌破了引发此次 SHIFT 的那个最低点 LL**。
2. **状态重置 (Trend Resumption)**: 此时，SHIFT 的高点被称为 **GL**，向下的突破被称为 **Liquidity break**。趋势状态 **重新恢复为空头 (Bearish)**，并继续向下寻找新的 LH 和 LL。

## 3. 多头趋势规则 (Bullish Trend)

### 3.1 HH (Higher High) 的确认条件
1. **突破**: 价格必须向上突破前一个 HH。
2. **回撤确认 (Retracement Confirmation)**: 价格在突破前一 HH 后，必须向下产生足够的回调。**程序化定义**：回撤深度必须 **>= `(前一HH - 前一HL) * 0.1`**。
3. **标记**: 当回调条件满足后，这波向上突破的绝对最高点被正式标记为新的 HH。

### 3.2 HL (Higher Low) 的确认条件
1. **相对高度**: HL 的顶点必须是最靠近前一个 HL，并且没有跌破它（即高于前一个 HL）。
2. **极值过滤 (Pivot Filter)**: 废除“必须在前一个HH(1)下方”的绝对高度限制。**程序化定义**：只要价格在形成新的 HH 之前，向下回调产生了一个局部的 ZigZag 极值低点（Pivot Low），且满足回撤确认深度的要求，即可作为候选 HL。这能有效应对单边暴涨行情中回调极浅的场景。
3. **突破确认**: 当价格再次向上突破最新的 HH 候选点（即发生 MSB）时，这段回调的最低点（Pivot Low）才被正式确认为 HL，并在此处画出 MSB Level 突破线。

### 3.3 多头 SHIFT 的确认条件
1. **反转突破**: 价格从 HH 顶点开始反转，并且最终向下跌破了**最近的一个确定的 HL** 顶点。
2. **结构成熟度**: 在发生 SHIFT 时，之前必须已经存在 **两对或以上** 的 (HL + HH) 结构。

### 3.4 多头 GL (Grab Liquidity)
1. **发生条件**: 在多头趋势发生 SHIFT（向下跌破 HL，理论上转空）之后，价格并没有继续下跌创造 LH/LL，而是向上 **突破了引发此次 SHIFT 的那个最高点 HH**。
2. **状态重置 (Trend Resumption)**: 此时，SHIFT 的低点被称为 **GL**，向上的突破被称为 **Liquidity break**。趋势状态 **重新恢复为多头 (Bullish)**，并继续向上寻找新的 HL 和 HH。

## 4. 初始状态 (Cold Start / Initialization)

因为所有的 MSB 和 SHIFT 都依赖于“前一个”高低点，图表最左侧的 K 线没有历史参考，必须进行冷启动初始化。

**程序化冷启动方案（全局极值法）：**
1. 选取图表最开始的 `N` 根 K 线（例如前 50 根或 100 根）。
2. 找出这 N 根 K 线中的 **绝对最高点 (Global Max)** 和 **绝对最低点 (Global Min)**。
3. 比较这两个极值发生的时间先后顺序：
   * 如果 **最低点先发生，最高点后发生**：我们假设初始趋势是 **多头 (Bullish)**。最低点作为初始的 `HL_0`，最高点作为初始的 `HH_0`。
   * 如果 **最高点先发生，最低点后发生**：我们假设初始趋势是 **空头 (Bearish)**。最高点作为初始的 `LH_0`，最低点作为初始的 `LL_0`。
4. 从第 `N+1` 根 K 线开始，正式启用上述的 MSB、回撤确认与 SHIFT 追踪逻辑。
